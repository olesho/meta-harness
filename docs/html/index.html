meta-harness

One uniform API over five terminal-based AI coding agents.

Claude Code, Codex, OpenCode, pi, and Cursor each ship as an interactive CLI with its own TUI, output format, and session log. meta-harness supervises any of them under a pseudoterminal and hides the differences — classifying output into a stable status taxonomy, detecting turn boundaries, parsing transcripts, and managing resumable sessions.

The layer cake

An acyclic stack, each layer a separate import

The core (chat) sits on three surface layers over a screen substrate, over a private async toolkit. Every arrow is a dependency; nothing lower reaches up. The one sanctioned leak: async re-exports only Context from the internal toolkit.

The life of a turn

From keystroke to classified reply

A send() writes keystrokes to the harness PTY. Bytes render into the screen; two pumps in the watcher turn each snapshot and status event into typed turn events; the chat state machine advances the pending turn to complete or errored.

Core concepts

Two ideas worth internalizing first

Two sessions, named from two sides

meta-harness mints its own id the instant you Open() — before the harness has revealed anything. The harness's own id surfaces later and gets captured; that's the one that resumes the session.

conv.sessionID()
Always present — keys the store & your records.
harness assigns
harnessSessionID
Empty until captured — resumes & locates the transcript.

History has a source

historyWithSource() tells you where turns came from. The transcript is authoritative — used only when the adapter has a reader and an id was captured — with a graceful fallback to the store.

Usage

Two programming models

A long-lived Conversation you drive turn by turn, or a one-shot call for a single prompt. Both take a Context first.

conversation.ts 
import { Open, newMemStore } from "meta-harness/chat"
import { Context } from "meta-harness/async"

const conv = await Open(Context.background(), {
  harness: "claude-code",
  binaryPath: "/usr/local/bin/claude",
  workingDir: process.cwd(),
  store: newMemStore(),
})

// one mutating op at a time
const release = await conv.acquireControl(ctx)
const turnID = await conv.send(ctx, "List the files.")
release()
// reply arrives via conv.events() → history()
await conv.close()
one-shot.ts 
import { runOneShot } from "meta-harness/oneshot"
import { Context } from "meta-harness/async"

// always bound a one-shot with a deadline
const { ctx, cancel } =
  Context.withDeadline(Context.background(), 120_000)

try {
  const reply = await runOneShot(ctx, {
    harness: "codex",
    binaryPath: "/usr/local/bin/codex",
    prompt: "Summarize README.md in one line.",
  })
  console.log(reply)
} finally {
  cancel()
}
